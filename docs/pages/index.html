DevContainer Wizard

A CLI tool that turns a simple YAML config into a ready-to-use devcontainer.json. Templates, interactive wizard, validation — no JSON wrestling.

⬇ Download latest ★ View on GitHub

Features

📝

YAML-first config

Write your dev container config in human-friendly YAML — no JSON, no escaping nightmares, comments allowed.

🧱

Built-in templates

Start from image, dockerfile, dockercompose, full, or golang templates with a single flag.

🧙

Interactive wizard

Run devcontainerwizard init -i to pick a template, set the container name, add features, and configure ports through guided prompts.

✅

Schema validation

Validates required fields, mutual exclusivity of image / build / dockerComposeFile, and enum values before writing any file.

⚡

Lifecycle commands

Supports onCreateCommand, postCreateCommand, postStartCommand, and postAttachCommand as a string or an array of strings.

🎨

Multi-editor support

Configure VS Code, JetBrains, Neovim, and Codespaces customizations — extensions, settings, plugins — all in one file.

🔌

Features & options

Install any devcontainer feature from containers.dev with per-feature options, including install order overrides.

🔒

Secrets support

Declare secrets with descriptions and defaults in YAML; they are written to devcontainer.json exactly as the spec requires.

📚

Built-in docs

Generate Markdown documentation for every config field with generate-docs, or browse it right in the terminal with show-docs.

Installation

go install github.com/lucasassuncao/devcontainerwizard@latest
# 1. Download the binary for your platform from the releases page
#    https://github.com/lucasassuncao/devcontainerwizard/releases

# 2. Add to your PATH, then run:
devcontainerwizard init -i
git clone https://github.com/lucasassuncao/devcontainerwizard.git
cd devcontainerwizard
go build -o devcontainerwizard .
./devcontainerwizard init -i

Quick start

Three steps from zero to a working .devcontainer/devcontainer.json.

devcontainerwizard init -i        # interactive wizard — pick template, name, features, ports
# or use a specific template directly:
devcontainerwizard init -t golang  # generate config from the Go template

# edit config.yaml as needed, then convert:
devcontainerwizard                 # writes .devcontainer/devcontainer.json

Configuration example

A typical config.yaml for a Go project.

name: my-go-project
image: "mcr.microsoft.com/devcontainers/go:latest"
workspaceFolder: "/workspaces/${localWorkspaceFolderBasename}"
remoteUser: vscode

features:
  ghcr.io/devcontainers/features/git:1:
    version: latest
  ghcr.io/devcontainers/features/go:1:
    version: "1.24"
    golangciLintVersion: latest

forwardPorts:
  - 8080

postCreateCommand:   # string or array
  - go mod download
  - go install github.com/air-verse/air@latest

customizations:
  vscode:
    extensions:
      - golang.go
      - eamodio.gitlens
    settings:
      go.formatTool: goimports
      go.lintTool: golangci-lint
      editor.formatOnSave: true

Commands

Command Description
devcontainerwizardConvert config.yaml to .devcontainer/devcontainer.json
devcontainerwizard -c path/to/config.yamlUse a custom config file path
devcontainerwizard -o path/to/dir/Write output to a custom directory
devcontainerwizard initCreate a config.yaml from the default (image) template
devcontainerwizard init -t golangCreate config.yaml from a specific template
devcontainerwizard init -iLaunch the interactive setup wizard
devcontainerwizard init -fOverwrite an existing config.yaml
devcontainerwizard generate-docsGenerate Markdown documentation for all config fields
devcontainerwizard show-docsBrowse documentation interactively in the terminal
